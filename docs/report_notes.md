# 基于 YOLO 的人脸情绪识别实践报告素材提纲

## 1. 需求分析

- 项目目标：实现一个可对图片和视频中的人脸进行情绪识别的演示系统。
- 应用场景：课堂演示、人机交互体验、安防/服务场景中的情绪状态辅助分析。
- 核心问题：如何在不自行训练模型的前提下，快速完成可运行、可测试、可截图的系统。
- 技术路线：YOLO 负责检测与情绪分类，DeepSORT 负责跨帧跟踪，OpenCV 负责读取、显示和保存结果。

## 2. 系统设计

- 模块划分：
  - 模型加载模块：加载人脸检测/分割模型和情绪分类模型。
  - 单帧处理模块：对每帧做检测、裁剪和情绪分类。
  - 跟踪与绘制模块：给检测目标分配 ID 并叠加情绪标签。
  - 输入输出模块：统一处理图片、视频、GUI 和命令行入口。
- 架构图建议：
  - 输入源 -> 模型加载 -> 单帧检测 -> DeepSORT 跟踪 -> 结果绘制 -> 图像/视频保存
- 流程图建议：
  - 读取输入 -> 判断图片/视频 -> 加载模型 -> 逐帧处理 -> 保存输出 -> 展示结果

## 3. 系统实现

- `function/model_loader.py`：负责加载 `models/yolo11n-seg.pt` 和 `models/best.pt`。
- `function/frame_processor.py`：对每个人脸区域执行情绪识别，并输出检测框与类别。
- `function/tracking.py`：根据 DeepSORT 输出绘制 `ID + 情绪标签`。
- `function/image_processor.py`：处理单张图片并保存结果。
- `function/video_processor.py`：处理视频流并导出结果视频。
- `main.py`：统一命令行入口，支持 `--source image|video --input --output`。
- `launch_gui.py`：GUI 启动入口，方便课堂演示。

## 4. 系统测试

- 测试环境：
  - Windows
  - Python 3.12
  - `numpy`, `opencv-python`, `ultralytics`, `deep_sort_realtime`
- 测试建议：
  - 图片测试：单人正脸、多人人脸、表情明显的人脸图片。
  - 视频测试：单人短视频、多人短视频。
  - 异常测试：空路径、错误文件类型、无可识别人脸的输入。
